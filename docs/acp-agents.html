Settings → Models → ACP Agents

ACP Agent 配置

Wisp 可作为 ACP Client，启动本机已安装的 Codex / Claude 等 ACP v1 Agent。 它与 HTTP 模型 profile 分开：Models 里的「添加模型」管 API， 「ACP Agents」管本地 stdio agent 进程。

开始配置 HTTP 模型配置 Markdown 文档
[ 001 ] HTTP vs ACP

两套后端，不要混用

Settings → Models 用两个分类切换：Models（HTTP API）与 ACP Agents（本地进程）。

HTTP 模型

Settings → Models → 添加模型。DeepSeek / OpenAI / Anthropic 等远程 API，走内置 Wisp Agent。

ACP Agent

Settings → Models → ACP Agents。本地 stdio 进程，Agent 自带会话、工具与鉴权。

不要填裸 CLI

不要把 codex / claude 直接填进 ACP。应使用官方 ACP 适配器。

凭证不进 SQLite

登录与 API key 由外部 Agent 自己管理；Wisp 只保存启动命令与参数。

[ 002 ] Where to click

入口在哪里

ACP 与「添加模型」同级，都在 Models 层级下。

1. Settings → Models

打开设置侧边栏的 Models，用顶部分类标签切换 Models / ACP Agents。

2. Add / Edit 同一套

右侧 Add model 或 Add ACP Agent 进入面包屑子页表单；点列表行即可编辑。

3. 聊天框快捷入口

模型菜单底部的 Add model / Add ACP Agent 打开同一套表单。

[ 003 ] Configure

填写与启用

先装好适配器，再在 Wisp 里保存、测试、选中。

字段 说明
Label 显示名，例如 Codex ACP
Command 只填可执行文件，不要把参数拼进这一行
Arguments 每个参数单独一行（Windows 用 npx.cmd 时尤需注意）

Save → Test Connection

成功表示进程能启动并完成 ACP initialize。

Authenticate（如有）

测试后若出现登录按钮，按 Agent 提示完成；凭证不写入 Wisp。

空会话中选用

新建空会话，在模型菜单选该 ACP Agent，再发消息。首条消息后锁定，换后端请开新会话。

[ 005 ] Session & limits

使用中注意

权限与配置

权限卡显示 Agent 返回的选项；会话级 model/mode 会显示在输入框上方。

科学工具

Wisp 会注入 MCP bridge，外部 Agent 可在项目目录内调用内置科学工具。

当前限制

仅本地 stdio；无远程 / WSL / SSH ACP；无应用内安装市场；编辑 Command/Arguments 后需新开会话。

[ 006 ] Troubleshooting

排错

现象 处理
Test Connection 立刻失败 Command 不在 PATH、Windows 应用 npx.cmd，或参数误写进 Command
认证失败 先在系统里完成 Codex / Claude 登录或 API key，再测
selection is locked 预期行为；换后端请新建空会话
profile or project path changed 改过启动参数或项目路径后需新开会话
